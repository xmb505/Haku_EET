# 电梯重量属性集成方案

## 目标

1. IO config 输入地址从 DB 格式改为 I 格式（消除多余的 `db_to_i` 转换层）
2. 为每部电梯的 `Car` 实体新增三态重量属性

---

## 任务 0：IO 输入地址改为 I 格式（前置重构）

### 动机

config 里输入地址存的是 `DB10.DBX0.0`，但 IO2HTTP 只认 I 地址（`I2.0`）。当前每读一个输入信号都要 `db_to_i(addr_input(...))`——43 处冗余调用。改成直接存 I 地址可删除整个 `db_to_i` 层。

输出地址不变——输出走 HTTP 直接写 DB11，不经过 bitmap，保留 `DB11.DBXx.y` 格式。

### 改动

**A) `config/io_config.yaml`**

- 删除 `db_to_i_offset: 2`
- 输入地址全部重写：`DB10.DBXx.y` → `I(x+2).y`

变化示例：

| 原地址 | 新地址 |
|--------|--------|
| `DB10.DBX0.0` | `I2.0` |
| `DB10.DBX2.2` | `I4.2` |
| `DB10.DBX27.6` | `I29.6` |

输出地址不变（保持 `DB11.DBXx.y` 格式）。

**B) `core/io_mapper.py`**

- 删除 `db_to_i()` 方法（不再需要）
- 删除 `i_to_db()` 方法（无调用方）
- 删除 `DB_ADDR_RE`、`I_ADDR_RE` 正则
- 删除 `db_to_i_offset` 属性
- 删除 `INPUT_DB_NUMBER` 常量
- `addr_input()` → 直接从 `_input_cache` 返回 I 地址
- `_i_to_signal` → 直接从 I 地址建索引（不需要转换）
- `_input_db_to_signal` → 键从 DB 地址变为 I 地址（对外不可见，不影响）

**C) 所有调用 `mapper.db_to_i(mapper.addr_input(...))` 的地方**

43 处全部简化为 `mapper.addr_input(...)`。

涉及文件：
- `core/executor.py`（~15 处）
- `core/app.py`（~10 处）
- `core/virtual_plc.py`（~6 处）
- `core/console.py`（~12 处）

**D) `tools/gen_io_config.py`**

- 更新生成逻辑：输入地址输出 I 格式，输出地址保持 DB 格式
- 删除 `db_to_i_offset` 生成

**E) `tests/test_io_mapper.py`**

- 删除 `db_to_i` / `i_to_db` 测试
- 更新地址测试用例为 I/Q 格式

### 不变的地方

- `addr_output()` → 仍然返回 `DB11.DBXx.y`，输出侧不变
- `io_client.py` → `set()` / `set_many()` 仍然收 DB 地址
- `ui.py` → 仍然调 `mapper.addr_output()` 返回 DB 地址

## 任务三：App 层改动（集成 weight_reader）

### 文件：`core/app.py`

#### 3.1 接受 weight_reader

```python
class App:
    def __init__(self, ..., weight_reader: Callable[[int], Awaitable[float]] | None = None):
        self.weight_reader = weight_reader
```

传 `None` 时跳过轮询（兼容现有测试和调试）。

#### 3.2 启动轮询任务

在 `start()` 中加入：

```python
if self.weight_reader is not None:
    for cid in self.car_ids:
        asyncio.create_task(self._weight_poll_loop(cid))
```

#### 3.3 轮询逻辑

```python
async def _weight_poll_loop(self, cid: int):
    cfg = self.config['elevator']['weight']
    while True:
        try:
            raw = await self.weight_reader(cid)
        except Exception:
            await asyncio.sleep(cfg['poll_interval'])
            continue

        car = self.cars[cid]
        old_state = car.weight_state

        if raw >= cfg['overload_threshold']:
            new_state = 2
        elif raw >= cfg['critical_threshold']:
            new_state = 1
        else:
            new_state = 0

        if new_state == old_state:
            await asyncio.sleep(cfg['poll_interval'])
            continue

        car.weight_state = new_state

        # 状态迁移动作：只有 state=2（超载）控制指示灯
        if new_state == 2:
            await self.ui[cid].set_full_load(True)
        elif old_state == 2:
            await self.ui[cid].set_full_load(False)

        labels = {0: '正常', 1: '满载', 2: '超载'}
        print(f'[weight] car{cid}: {labels[old_state]} → {labels[new_state]} ({raw}kg)')

        await asyncio.sleep(cfg['poll_interval'])
```

#### 3.4 派车时检查满载

在 `_dispatch_hall_call()` 的顺路分支中：

```python
# 修改前
if same_dir_pass:
    candidates.append((0, abs(floor - pos), cid))

# 修改后
if same_dir_pass:
    if car.weight_state == 1:
        continue  # 满载运行中，不派顺路外呼
    candidates.append((0, abs(floor - pos), cid))
```

> 满载（state=1）只跳过**顺路（priority 0）**分支。空闲车（priority 1）不受影响。

## 任务四：Passenger 层改动（关门时检查超载）

### 文件：`core/passenger.py`

#### 4.1 关门 cron 检查超载

```python
if self._app.cars[cid].weight_state == 2:
```

```python
    if self._app.cars[cid].weight_state == 2:
        print(f'[door] car{cid}: 超载，拒绝关门按钮')
        return
```

## 控制流总结

```
weight_reader(car_id) → float
     │
     ▼
App._weight_poll_loop (每 100ms)
     │
     ├── raw < critical        → weight_state = 0
     ├── critical ≤ raw < overload → weight_state = 1
     └── raw ≥ overload        → weight_state = 2, 亮 full_load_indicator
     │
     ▼ 派车时
_dispatch_hall_call(): 顺路分支检查 weight_state==1 → 跳过
     │
     ▼ 关门时
_schedule_close_door_cron_job(): 检查 weight_state==2 → 跳过
on_door_button(close): 检查 weight_state==2 → 跳过
```

## 不变的文件

| 文件 | 原因 |
|------|------|
| `io_client.py` | weight 不走 bitmap，输出地址格式不变 |
| `executor.py` | weight 不涉及动作执行（仅 IO 地址简化） |
| `controllers.py` | 关门检查放在大脑层 |
| `ui.py` | `set_full_load()` 已存在，复用 |
