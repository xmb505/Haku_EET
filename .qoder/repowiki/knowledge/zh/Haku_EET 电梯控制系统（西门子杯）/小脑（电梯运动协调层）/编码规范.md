- 所有对 PLC 的输出写统一走 `io_write.set_many({mapper.addr_output(sig, car_id): value})`，禁止直接调用底层 set()，以保证多车写缓冲合并与 tick flush 顺序可控。
- IO 事件入口 `on_io_event` 先 `self.io.observe_input(event.i_addr, event.bit)` 同步 cache，再用 `mapper.lookup_signal_by_i` 反查 (car_id, signal_name) 做归属过滤，确保跨轿厢事件不会串扰。
- 长寿命瞬态状态（如 `_level_seek_active`、`_auto_seek_active`、`_relevel_future`）在急停 `_emergency_stop` 中集中清零，保证 FAULT 后不再被残留标志唤醒。
- 非 NOOP 动作进入 `_start_action` 时一律退出站点吸附保持模式（清 `_level_seek_active` 和 `_relevel_future`），NOOP 除外——避免空闲 NOOP 反复踢掉 hold。
- App 暴露给 console 的高层 API（call_internal/change_internal/fireman/reset/manual_*）统一以 `car_id | None = None` 参数配合 `cid = car_id if car_id is not None else self.current_car_id` 选择目标轿厢。
